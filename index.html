配置Nginx前端 + Apache后端服务器环境

发表于 2016-06-21

nginx处理静态内容是把好手，apache虽然占用内存多了点，性能上稍逊，但一直比较稳健。倒是nginx的FastCGI有时候会出现502 Bad Gateway错误。一个可选的方法是nginx做前端代理，处理静态内容，动态请求统统转发给后端apache。
本文就是实现Nginx作为前端，apache作为后端。当用户访问的是80端口的nginx，然后nginx将静态内容留给自己，其余的转发给非80端口的apache，apache处理完毕后再回传给nginx。例如这里我分别访问 www.test.com/a.html,www.test.com/info.php。nginx因为做前面，nginx的访问日志应该都有a.html,info.php的记录；而apache作为后端，只处理php请求，应该只有info.php的访问记录。

整理：

  • 安装nginx

  • 修改配置

  • 期间安装了php加速

    安装php加速器eaccelerator和php-pecl-memcached模块，提高性能：

    #

    The default server

    #
    server {

    listen      80;
server_name jkkj.com;

index  index.html index.htm index.php;
root   /var/www/html;
access_log   /var/log/nginx/jkkj.com.log;
# nginx找不到文件时，转发请求给后端Apache
error_page 404 @proxy;

error_page   500 502 503 504  /50x.html;
location = /50x.html {
    root   /usr/share/nginx/html;
}
# 动态文件.php请求转发给后端Apache
location ~ .php$ {
    # 向后端服务器发起请求时添加指定的header头信息
    proxy_set_header Host $http_host;
    # 向后端服务器发送真实 IP
    proxy_set_header X-Real-IP $remote_addr;
    # 让后端如php能直接通过变量获取真实IP
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_pass   http://127.0.0.1:8080;
}
# nginx找不到文件时，转发请求给后端Apache
location @proxy {
    proxy_set_header Host $http_host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_pass http://127.0.0.1:8080;
}

Listen 127.0.0.1:8080  //站点
Listen 45.xxx.xxx.xxx:8090 //资源(后面会讲到)


<VirtualHost 45.xx.xxx.xxx:8080>
        ServerAdmin root@localhost
        DocumentRoot /var/www/html
        ServerName www.jkkj.com
        ErrorLog logs/error_log.jkkj.com
        CustomLog logs/custom.log common
</VirtualHost>

<VirtualHost 45.xx.xxx.xxx:8090>
        ServerAdmin root@localhost
        DocumentRoot /var/www/html/Public
        ServerName img.fdzzj.com
        ErrorLog logs/error_log.fdzzj.com
        CustomLog logs/custom.log common
</VirtualHost>

server {
    listen      80;
    server_name  *.fdzzj.com;

    root   /var/www/html/Public;
    index  index.html index.htm;

    error_page  404  @proxy ;

    location / {
        proxy_pass http://45.xx.xxx.xxx:8090;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # nginx找不到文件时，转发请求给主站的后端Apache
    location @proxy {
        proxy_set_header Host $http_host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_pass http://127.0.0.1:8080;
    }
}

/etc/init.d/httpd restart
/etc/init.d/nginx restart

只做一件事

发表于 2016-06-21

只做一件事,做好这件事

随笔记下发生在自己身边的这四件事:

  • 最早的时候就读到过一篇关于”做好一件事”的文章,当时读有所感就马上分享给了我们公司的群.

    那篇的内容主要说人的一生只要做好一件事情就足够了,如我们书本里耳熟能详的”李时珍”等等,他们用他们的一生只做了一件事情,他们也许想做更多,但大部分都专注于这件事,认为还可以更好.”还可以更好”,更多的时候我们会满足于已经完成的.

    或许那时我也懂了明白了一些,感觉也灌了心灵鸡汤.

  • 之后的我还是老样子,使用了github.com之后,发现原来世界上有这么多程序员在不断创新的同时,一直维护自己的某个小开源项目.他的开源项目或许刚开始很搓,但在一直维护下,一个小小的功能被完善得让人觉得他就是标准.

  • 在上次和中华讨论的代码优化上,一致认为了某位前辈的经验,一个函数只做一件事情,一个函数不超过20行.真的去这么做之后的确发现不少的好处.代码的维护更方便,代码的重复利用更有效.
  • 最近网站的框架升级,有许多地方要改.我在替换新老代码时也奉行了只做一件事情的原则.在替换某个字符串时,绝不因为看到另一个顺手也改下,每次的改动只做一件事,并记录下这件事的文字说明.如果按以前老习惯去,我想我一定会做许多重复的事情,或许自己也会被搞疯掉.

ubuntu14.04安装nvm,nodejs,express

发表于 2016-06-21

安装 curl和git

sudo apt-get install curl git

下载并运行NVM安装sh

wget https://raw.github.com/creationix/nvm/master/install.sh
sh ./install.sh

安装之后输入nvm还是提示没有这时候需要执行

source ~/.nvm/nvm.sh

将source /root/.nvm/nvm.sh写入~/.bashrc或者其启动脚本中，这样在系统启动的时候会自动执行这条指令。开机就可以使用nvm了；

安装Node v4.2.2

nvm install v4.2.2

npm设置淘宝镜像

淘宝npm镜像是国内最大的，访问速度快的npm镜像，设置方法如下：
编辑~/.npmrc文件（没有自己创建）
加入以下配置信息：

registry = http://registry.npm.taobao.org

搞定！

npm如何升级自身

npm install npm -g
安装Expres和Express-generator（生成Express工程）
npm install -g express
npm install -g express-generator

如此就已经基本安装完成了
